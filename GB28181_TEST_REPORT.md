## 四、发现问题与修复

### 4.1 已修复问题

#### 问题 1: 测试脚本 REGISTER Request-URI 格式错误

- **影响**: 注册测试失败（返回 401）
- **原因**: Request-URI 使用 `sip:host:port` 而非 `sip:deviceId@host:port`
- **修复**: 修改测试脚本，使用正确的 Request-URI 格式
- **文件**: `test_whitelist_blacklist.py`, `test_new_device_auth.py`

#### 问题 2: 数据库配置错误

- **影响**: Catalog 查询 API 返回 404
- **原因**: DbUtil.cpp 中使用错误的数据库名 `gb_service2022`
- **修复**: 修改为正确的数据库名 `gb28181`
- **文件**: `backend/src/infra/DbUtil.cpp`

### 4.2 待修复问题

#### 问题 1: PJSIP Catalog 查询发送崩溃

- **影响**: 无法发送 SIP Catalog 查询消息
- **现象**: 
  ```
  gb_service: ../src/pjsip/sip_util_statefull.c:98: 
  pjsip_endpt_send_request: Assertion 'mod_stateful_util.id != -1' failed.
  ```
- **原因**: 调用 `pjsip_endpt_send_request` 时需要状态模块，但模块未初始化
- **建议修复**: 
  - 使用 `pjsip_endpt_send_request_stateless` 发送无状态请求
  - 或确保正确初始化 PJSIP 状态模块
- **优先级**: 高
