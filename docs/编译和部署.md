# 编译和部署指南

本文档用于说明 `gb_service2022` 的常用编译、运行验证与部署流程。

## 1. 环境准备

### 1.1 基础要求

- Linux（推荐 Ubuntu 20.04+）
- CMake 3.14+
- GCC/G++（支持 C++17）
- Node.js 18+（前端构建）
- PostgreSQL、Nginx（部署机）

### 1.2 拉取代码与子模块

```bash
git clone --recurse-submodules <your-repo-url>
cd gb_service2022
```

如果已 clone 但未拉子模块：

```bash
git submodule update --init --recursive
```

### 1.3 构建 PJSIP（首次或缺失时）

后端依赖 `third_party/pjproject/install`，若目录不存在先执行：

```bash
./scripts/build_pjproject.sh
```

## 2. 本地编译

### 2.1 编译后端

```bash
mkdir -p backend/build
cmake -S backend -B backend/build
cmake --build backend/build
```

可选：执行测试

```bash
cd backend/build && ctest --output-on-failure
```

### 2.2 构建前端

```bash
cd frontend
npm install
npm run build
```

构建产物：`frontend/dist/`

## 3. 本地运行与验收

### 3.1 启动后端（开发模式）

```bash
pkill -x gb_service 2>/dev/null || true
cd backend/build
nohup ./gb_service >> /tmp/gb_service.log 2>&1 &
```

### 3.2 健康检查

```bash
curl -s http://127.0.0.1:8080/api/health
```

期望返回包含：`{"status":"ok"}`

### 3.3 一键自测（可选）

```bash
./scripts/rebuild-and-run.sh
```

该脚本会顺序执行：后端编译 -> 单测 -> 重启 `gb_service` -> 前端构建。

## 4. 部署流程（服务器）

本仓库提供 `deploy/install.sh` 作为一键部署入口，建议按下面步骤准备部署包后执行。

### 4.1 在构建机准备产物

先完成第 2 节编译，然后整理部署目录（对齐 `deploy/install.sh` 期望结构）：

```bash
mkdir -p deploy/backend deploy/frontend/dist
cp backend/build/gb_service deploy/backend/gb_service
rm -rf deploy/frontend/dist/*
cp -r frontend/dist/* deploy/frontend/dist/
```

打包：

```bash
tar -czf gb28181-deploy.tar.gz deploy
```

### 4.2 上传并安装

```bash
scp gb28181-deploy.tar.gz <user>@<server-ip>:/tmp/
ssh <user>@<server-ip>
cd /tmp
tar -xzf gb28181-deploy.tar.gz
cd deploy
sudo ./install.sh
```

默认安装目录：`/opt/gb_service2022`

### 4.3 数据库配置文件（可选）

后端会优先读取 `config/gb_service.ini`（部署后通常放在 `/opt/gb_service2022/config/gb_service.ini`）。

可参考仓库模板：

```bash
cp config/gb_service.ini.example config/gb_service.ini
```

示例内容：

```ini
host=/var/run/postgresql
port=5432
dbname=gb28181
user=user
password=
```

说明：
- 必填：`dbname`、`user`
- 可选：`host`、`port`、`password`
- 若文件缺失或解析失败，服务会回退默认：`host=/var/run/postgresql dbname=gb28181 user=user`

## 5. 部署后验证

```bash
curl -s http://127.0.0.1:8080/api/health
curl -s http://127.0.0.1/api/health
```

浏览器访问：

- `http://<server-ip>/`（前端）

## 6. 服务运维命令

```bash
sudo systemctl status gb_service
sudo systemctl restart gb_service
sudo journalctl -u gb_service -f
sudo tail -f /var/log/gb_service.log
```

Nginx 检查：

```bash
sudo nginx -t
sudo systemctl reload nginx
```

## 7. 常见问题

- 启动失败：优先看 `journalctl -u gb_service -n 100`
- 健康检查失败：确认 `8080` 端口监听与数据库可连通
- 前端空白：确认已执行 `npm run build` 且部署目录下静态资源完整
