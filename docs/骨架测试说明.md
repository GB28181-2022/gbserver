# 项目骨架测试说明

骨架阶段验收标准：以下步骤全部通过后，方可进入下一小周期（如用户登录）。

## 1. 后端 C++ 骨架

### 1.1 编译

```bash
cd /home/user/coder/gb_service2022/backend
mkdir -p build && cd build
cmake ..
make
```

**通过标准**：无编译错误，生成可执行文件 `gb_service`。

### 1.2 运行与健康检查

```bash
# 在 backend/build 目录下
./gb_service
```

另开终端：

```bash
curl -s http://127.0.0.1:8080/api/health
```

**通过标准**：返回 `{"status":"ok"}`，且 `curl -s -o /dev/null -w "%{http_code}" http://127.0.0.1:8080/api/health` 为 `200`。

### 1.3 非健康路径返回 404

```bash
curl -s -o /dev/null -w "%{http_code}" http://127.0.0.1:8080/api/other
```

**通过标准**：返回 `404`。

---

## 2. 前端 Vue 骨架

**环境**：Node 14+ 可构建（当前使用 Vite 4）；若使用 Node 18+ 可升级为 Vite 5。

### 2.1 安装依赖与构建

```bash
cd /home/user/coder/gb_service2022/frontend
npm install
npm run build
```

**通过标准**：无报错，且存在 `frontend/dist/index.html` 及 dist 内 js/css 资源。

### 2.2 本地预览（可选）

```bash
cd /home/user/coder/gb_service2022/frontend
npm run dev
```

浏览器访问 http://localhost:5173 ，页面显示「国标服务器系统」且后端健康状态。（需同时运行后端 `gb_service`，且 vite 已配置 `/api` 代理到 8080。）

---

## 3. 集成：Nginx + 后端 + 前端静态

### 3.1 启动后端

```bash
cd /home/user/coder/gb_service2022/backend/build
./gb_service
```

保持运行。

### 3.2 确认 Nginx 已加载国标配置

Nginx 已配置：80 → 前端 `frontend/dist`，`/api/` → 127.0.0.1:8080。

### 3.3 经 Nginx 访问

```bash
curl -s http://127.0.0.1/api/health
```

**通过标准**：返回 `{"status":"ok"}`。

浏览器访问 http://127.0.0.1/ （或本机 IP）  
**通过标准**：打开「国标服务器系统」页面，且页面显示后端健康为「ok」（或等价成功状态）。
