# 编码阶段就绪检核

> 检核目的：判断当前需求、功能清单与架构设计是否足以支撑进入代码编码阶段。  
> 依据文档：《国标服务器系统需求规格说明书》《功能清单》《架构设计》《架构设计检核》。

---

## 一、结论摘要

| 维度 | 结论 | 说明 |
|------|------|------|
| **需求与功能** | ✅ 具备 | 需求规格、功能清单、MVP 范围明确，可据此拆分任务与验收 |
| **架构与模块** | ✅ 具备 | 模块划分、依赖、数据流、部署、技术选型已定，可指导工程结构 |
| **接口与集成** | ⚠️ 基本具备 | 模块间接口、ZLM/PJSIP 集成要点已写；REST API 仅清单级，编码时需边做边补 |
| **可编码的最小闭环** | ✅ 具备 | 可从「工程骨架 + 用户登录 + 配置管理」切入，再扩展 SIP/设备/流媒体 |

**总体结论**：**可以进入编码阶段**。建议采用「MVP 分阶段实现」：先搭工程与基础设施（Nginx、8080、PostgreSQL、用户登录），再实现 SIP 信令与设备管理，最后接 ZLM 与点播/回放。REST API、日志、错误策略等在编码/详细设计时补充即可。

## 四、编码前可做与可延后的内容

### 4.1 编码阶段可直接落地（无需再补文档）

- 工程结构：C++ 后端（按模块分目录，集成 ZLToolKit 基础设施）、Vue + Element Plus 前端、Nginx 配置
- 用户与鉴权：登录/登出/修改密码、token 或 session、PostgreSQL 用户表
- 配置管理：本地国标信息、流媒体配置的 CRUD、存 PostgreSQL
- SIP 信令：PJSIP 初始化、Register/心跳处理、与配置管理/设备管理对接
- 设备管理：设备树结构、在线状态、手动添加、黑白名单与鉴权逻辑
- ZLM 协调：openRtpServer、closeRtpServer、startSendRtp、startRecord；Web Hook 接口与路由
- 录像元数据：设备/通道/时间/流 ID 存库，历史检索查库 + ZLM 播放 URL

### 4.2 建议在编码或详细设计时补充（不阻塞开工）

| 内容 | 当前状态 | 建议 |

## 五、推荐的编码切入顺序

1. **基础设施**
   - 仓库结构（C++ 工程、Vue 工程、Nginx 配置目录）
   - 编译与运行：C++ 依赖（PJSIP、libpq 等）、Vue 构建、Nginx 反向代理到 8080
   - PostgreSQL：建库、用户表、配置表（可先最小集）

2. **用户与鉴权（AUTH-001～003）**
   - 后端：登录/登出/修改密码 API，token 生成与校验中间件
   - 前端：登录页、登出、修改密码入口
   - 未登录访问 8080 除登录接口外均 401，前端跳转登录页

3. **配置管理（CFG-001、CFG-002）**
   - 后端：本地国标信息、流媒体配置的 CRUD API，读入后供 SIP/ZLM 使用
   - 前端：配置页表单与保存

4. **SIP 信令 + 设备管理（SVR-101～110）**
   - PJSIP 服务端：监听 5060、Register/心跳、鉴权（调配置管理）、设备树更新
   - 设备管理：设备树、在线状态、手动添加、黑白名单与独立密码逻辑
   - 前端：设备树页、在线状态、手动添加与黑白名单配置

5. **流媒体协调 + 点播/回放（SVR-201～202、SVR-301～303）**
   - ZLM：openRtpServer、closeRtpServer、startRecord、Web Hook 注册与处理（on_stream_changed、on_stream_none_reader）
   - 点播/回放流程按 5.2/5.3/9.3 实现；播放 URL 由后端 API 提供
   - 前端：监控页、点播、回放、jessibuca 播放

6. **告警、PTZ、客户端模式等**
   - 按功能清单优先级（P0 告警、PTZ；P1 客户端编组/接入/媒体推送）逐步加

## 六、风险与前提

| 风险/前提 | 说明 | 建议 |
|-----------|------|------|
| REST API 未定稿 | 前后端可能对接口路径/字段有歧义 | 每做一个功能前用一页文档或注释约定 API，再实现 |
| PJSIP/ZLM 上手成本 | 需熟悉 GB28181 与 ZLM API | 先做「仅 Register + 设备树」与「仅 openRtpServer + 一路点播」的最小验证，再扩展 |
| 双模式同时启用 | 端口与 ZLM 共用已说明，实现时需测试 | 在 7.2 基础上做一次部署检查清单，避免遗漏端口或 Hook 地址 |
